# AI vs. Human Text Detection

Project 1 — Intro to LLMs. Detects whether a piece of text was written by a
human or generated by AI, using 6 trained classifiers and a Streamlit web app.

## What's in here

```
ai_human_detection_project/
├── app.py                      # Streamlit web app (Part 2)
├── requirements.txt
├── README.md
├── .gitignore
├── utils/
│   ├── text_features.py        # shared linguistic-feature extractor (used by both notebook & app)
│   └── remote_models.py        # downloads large model artifacts from the GitHub Release
├── models/                      # trained model artifacts (see "Models" below)
├── data/
│   ├── training_data/train.csv
│   └── test_data/test.csv
├── scripts/                     # pipeline scripts (feature engineering, training, tuning)
└── notebooks/
    └── project1_notebook.ipynb  # Part 1 — all 4 required sections
```

## Setup

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## Models

**Committed directly to the repo** (small enough for normal git):
- `models/tfidf_vectorizer.pkl`, `models/linguistic_scaler.pkl`
- `models/svm_model.pkl` — SVM, 98.0% accuracy
- `models/decision_tree_model.pkl` — Decision Tree, 91.5% accuracy
- `models/adaboost_model.pkl` — AdaBoost, 94.8% accuracy
- `models/fnn_sklearn_model.pkl` — feedforward neural network (sklearn `MLPClassifier`), 99.1% accuracy — fast reference FNN

**Hosted as [GitHub Release assets](https://github.com/LeoLakshman/ai_human_detection_project/releases/tag/v1.0.0)**
instead of committed directly — the LSTM and CNN models are each ~30MB, over
GitHub's comfortable size limit for a normal push:
- `fnn_model.h5`, `lstm_model.h5`, `cnn_model.h5` — trained in Google Colab (GPU)
- `word2vec.model`, `tokenizer.json` — embeddings/tokenizer needed by those models

`app.py` downloads these automatically on first run via `utils/remote_models.py`
and caches them in `models/` afterward, so you don't need to fetch them by hand.
If you re-train and re-upload new versions, bump the tag in
`utils/remote_models.py` (`GITHUB_RELEASE_BASE_URL`) to match.

## Running the app

```bash
streamlit run app.py
```
The model selector only lists models that successfully loaded (or downloaded)
— so it still works even if a release asset is temporarily unavailable, just
with a smaller set of models to choose from.

## Notebook

`notebooks/project1_notebook.ipynb` contains all four required sections.
Sections 1–2 and the classical-ML half of Section 3 (SVM, Decision Tree,
AdaBoost, sklearn FNN) run end-to-end locally. The Keras FNN/LSTM/CNN were
trained in Colab (`scripts/03_deep_learning_RUN_LOCALLY.py` has the training
code) — their results are merged into the notebook's Section 4 comparison
table via `deep_learning_results.json`.

## Dataset

`train_data_with_labels.xlsx` → 8,176 documents, perfectly balanced
(4,088 human / 4,088 AI), label 0 = human, label 1 = AI. Split 80/20 stratified
into `data/training_data/train.csv` and `data/test_data/test.csv`.

## Notes for the write-up / demo video

- Feature comparison (full dataset): TF-IDF alone reached ~97.9% accuracy;
  linguistic features alone ~82.5%; combining them didn't improve on TF-IDF
  alone for the linear model — i.e. TF-IDF is already capturing most of the
  separable signal for this dataset.
- "SVM" is implemented as a calibrated `LinearSVC` (linear kernel) — standard
  practice for high-dimensional sparse TF-IDF features, and much faster than
  a kernel SVM at this feature size with no accuracy loss.
- The dataset's texts are long, formal/academic-style passages, so accuracy
  likely drops on short, casual, or out-of-domain text — worth demonstrating
  live in the app and discussing as a limitation.
